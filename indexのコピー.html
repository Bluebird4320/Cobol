🏗️ 1. COBOLプログラムの骨格 ✅ ▼

COBOLは4つのDIVISION（部門）が決まった順番で並ぶ構造です。まずこの骨格を体に染み込ませましょう。

📌 基本骨格

       IDENTIFICATION DIVISION.
       PROGRAM-ID. PROG001.
      *プログラムの名刺のような部分

       ENVIRONMENT DIVISION.
       INPUT-OUTPUT SECTION.
       FILE-CONTROL.
           SELECT 入力ファイル ASSIGN TO "INPUT.DAT"
               ORGANIZATION IS SEQUENTIAL.
      *ファイルと装置の関連付け

       DATA DIVISION.
       FILE SECTION.
       FD  入力ファイル.
       01  入力レコード         PIC X(80).

       WORKING-STORAGE SECTION.
       01  WS-COUNTER          PIC 9(5) VALUE ZEROS.
      *変数・作業域の宣言

       PROCEDURE DIVISION.
      *━━ここから実際の処理━━
           OPEN INPUT 入力ファイル
           PERFORM 読込処理 UNTIL WS-EOF = "Y"
           CLOSE 入力ファイル
           STOP RUN.

       読込処理.
           READ 入力ファイル
               AT END MOVE "Y" TO WS-EOF
           END-READ.

📌 カラム規則（固定形式）を必ず守る！

列番号: 1234567890123456789...72
              |||||
              |||B領域（12〜72列）: 命令文
              ||A領域（8〜11列）: 段落名・DIVISION名
              |7列: * でコメント行
              1〜6列: 行番号（現在は省略が多い）
  • A領域に書くもの：DIVISION名、SECTION名、段落名、FD、01レベル
  • B領域に書くもの：命令文、02〜49レベルのデータ定義
  • 73列以降は無視される（コメントに使う現場もある）

📌 実務ポイント

  • PROGRAM-IDは8文字以内が基本（現場規約による）
  • コメントは*を7列目に書く。*>は自由形式コメント
  • STOP RUN は必ず書く。ないとプログラムが暴走する
📊 2. データ定義とPIC句 ✅ ▼

COBOLの最大の特徴はデータ定義が詳細なこと。PIC句でデータの型・桁数・初期値まで宣言します。

📌 PIC句の指定子一覧

9  : 数字1桁（0〜9）
X  : 英数字1文字（文字列）
A  : 英字1文字（A〜Z, a〜z, スペース）
V  : 仮想小数点（実際のピリオドではない！）
S  : 符号付き（マイナスを扱う）
Z  : 先頭ゼロを空白に抑制（帳票出力向け）
.  : 実小数点（編集項目）
,  : カンマ挿入（編集項目）
+/-: 符号挿入（編集項目）
¥  : 円マーク挿入（編集項目）

📌 よく使うPIC句の例

*【数値系】
01 WS-SHIYO-RYYO  PIC 9(6).      *999999 最大999999
01 WS-TANKA       PIC 9(4)V9(2). *9999.99（仮想小数）
01 WS-SAHEN       PIC S9(7).     *符号付き7桁

*【文字系】
01 WS-KOKYAKU-ID  PIC X(8).      *"A0000001"
01 WS-KOKYAKU-NM  PIC X(20).     *氏名20文字

*【編集項目（表示・印刷用）】
01 WS-DISP-RYOKIN PIC ZZZ,ZZ9.   *  1,234（ゼロ抑制+カンマ）
01 WS-DISP-TANKA  PIC ¥¥¥,¥¥9.  *¥1,234（円マーク）

📌 レベル番号とグループ項目

*01が最上位。数字が大きいほど下位項目
01 料金レコード.
   05 KO-ID         PIC X(8).
   05 KO-NAME       PIC X(20).
   05 使用量グループ.
      10 当月使用量  PIC 9(6).
      10 前月使用量  PIC 9(6).
      10 増減量      PIC S9(6).   *符号付き（減少もある）
   05 KO-RYOKIN     PIC 9(9).
   05 KO-ZEINUKI    PIC 9(9).

*グループ項目は MOVE で丸ごとコピーできる
MOVE 使用量グループ OF レコードA TO 使用量グループ OF レコードB

📌 VALUE句（初期値）

01 WS-EOF-FLG    PIC X(1)  VALUE "N".
01 WS-COUNTER    PIC 9(5)  VALUE ZEROS.   *= VALUE 0
01 WS-TITLE      PIC X(10) VALUE SPACES.  *スペース初期化
01 WS-MAX        PIC 9(3)  VALUE 100.
01 WS-NAME       PIC X(8)  VALUE "TANAKA".

📌 特殊レベル番号

*77レベル：グループに属さない独立作業域
77 WS-WORK-NUM   PIC 9(9).
77 WS-WORK-STR   PIC X(50).

*88レベル：条件名（フラグ管理が格段に楽になる！）
01 WS-STATUS     PIC X(1).
   88 処理中      VALUE "P".
   88 完了        VALUE "C".
   88 エラー      VALUE "E".

*使い方
SET 処理中 TO TRUE       *WS-STATUS = "P" と同じ
IF 完了                  *WS-STATUS = "C" と同じ
    PERFORM 完了処理
END-IF

📌 実務ポイント

  • 仮想小数点Vは計算に使い、実小数点.は表示専用
  • 編集項目（Z, ¥など）はMOVEで受け取るだけ、計算には使えない
  • 88レベルを活用するとコードの可読性が大幅に上がる
🔀 3. MOVE命令とデータ転送 ✅ ▼

MOVEはCOBOLで最もよく使う命令。型変換の挙動を正確に理解することが実務の鍵です。

📌 基本形と初期化

MOVE 0         TO WS-COUNTER      *数値を0に
MOVE ZEROS     TO WS-COUNTER      *同上（複数項目に一度に使える）
MOVE SPACES    TO WS-NAME         *文字をスペースに
MOVE HIGH-VALUES TO WS-KEY        *X'FF'で埋める（最大値比較に使う）
MOVE LOW-VALUES  TO WS-KEY        *X'00'で埋める（最小値比較に使う）
MOVE WS-A      TO WS-B            *変数から変数

📌 型変換の挙動（重要！）

*【数値→数値】右詰め、左ゼロ埋め
WS-A PIC 9(5) VALUE 123.
WS-B PIC 9(8).
MOVE WS-A TO WS-B    → WS-B = 00000123

*【文字→文字】左詰め、右スペース埋め
WS-C PIC X(3) VALUE "AB".
WS-D PIC X(6).
MOVE WS-C TO WS-D    → WS-D = "AB    "（右4スペース）

*【文字→数値】数字文字のみ転送（非数字は0）
*【数値→文字】数字を右詰め文字列に変換

📌 MOVE CORRESPONDING（対応項目一括転送）

01 旧レコード.
   05 KO-ID    PIC X(8).
   05 KO-NAME  PIC X(20).
   05 KO-ADDR  PIC X(50).

01 新レコード.
   05 KO-ID    PIC X(8).
   05 KO-NAME  PIC X(20).
   05 KO-TEL   PIC X(15).  *旧にはない項目

*名前が同じ項目だけ一括転送！
MOVE CORRESPONDING 旧レコード TO 新レコード
*→ KO-IDとKO-NAMEだけ転送、KO-ADDRとKO-TELは無変化

📌 よくあるミス

*❌ 桁あふれに注意！（上位桁が切れる）
WS-A PIC 9(3) VALUE 999.
WS-B PIC 9(2).
MOVE WS-A TO WS-B  → WS-B = 99（上位1桁が消える！）

*❌ 仮想小数点のずれ
WS-A PIC 9(4)V9(2) VALUE 1234.56.
WS-B PIC 9(6).
MOVE WS-A TO WS-B  → WS-B = 123456（小数点位置で合わせる）

*✅ 対処：COMPUTE や桁数を合わせて設計する
🔢 4. 算術命令（COMPUTE・ADD等） ✅ ▼

COBOLの算術命令は2種類。COMPUTE（式で書く）とADD/SUBTRACT/MULTIPLY/DIVIDE（動詞で書く）。

📌 COMPUTE（式形式・推奨）

*基本形
COMPUTE WS-RYOKIN = WS-TANKA * WS-SURYO

*複合式（演算子: + - * / **）
COMPUTE WS-ZEIKOMI = WS-RYOKIN * 1.1

*べき乗
COMPUTE WS-COMPOUND = WS-GANKIN * (1 + WS-RISHI) ** WS-YEARS

*ROUNDED（四捨五入）
COMPUTE WS-RYOKIN ROUNDED = WS-TANKA * WS-SURYO

*ON SIZE ERROR（桁あふれ検知）
COMPUTE WS-RESULT = WS-A * WS-B
    ON SIZE ERROR
        MOVE "E" TO WS-STATUS
        PERFORM エラー処理
END-COMPUTE

📌 ADD / SUBTRACT / MULTIPLY / DIVIDE

*ADD
ADD WS-RYOKIN    TO WS-GOUKEI      *WS-GOUKEI += WS-RYOKIN
ADD WS-A WS-B   TO WS-C            *WS-C += WS-A + WS-B
ADD WS-A TO WS-B GIVING WS-C       *WS-C = WS-B + WS-A（WS-Bは変わらない）

*SUBTRACT
SUBTRACT WS-WARIBIKI FROM WS-RYOKIN     *WS-RYOKIN -= WS-WARIBIKI
SUBTRACT WS-A FROM WS-B GIVING WS-C    *WS-C = WS-B - WS-A

*MULTIPLY
MULTIPLY 1.1 BY WS-RYOKIN              *WS-RYOKIN *= 1.1
MULTIPLY WS-A BY WS-B GIVING WS-C      *WS-C = WS-A * WS-B

*DIVIDE
DIVIDE WS-GOUKEI BY 12 GIVING WS-TSUKI REMAINDER WS-AMARI
*WS-TSUKI = WS-GOUKEI ÷ 12（余りはWS-AMARIへ）

📌 電気料金計算の実例

*段階料金計算（実務パターン）
PROCEDURE DIVISION.
    EVALUATE TRUE
        WHEN WS-SHIYO <= 120
            COMPUTE WS-RYOKIN ROUNDED
                = WS-SHIYO * 17.78
        WHEN WS-SHIYO <= 300
            COMPUTE WS-RYOKIN ROUNDED
                = 120 * 17.78
                + (WS-SHIYO - 120) * 22.86
        WHEN OTHER
            COMPUTE WS-RYOKIN ROUNDED
                = 120 * 17.78
                + 180 * 22.86
                + (WS-SHIYO - 300) * 25.72
    END-EVALUATE
    COMPUTE WS-ZEIKOMI ROUNDED = WS-RYOKIN * 1.10.

📌 実務ポイント

  • 金額計算は必ずROUNDEDを付けて端数処理を明示
  • 桁あふれはON SIZE ERRORで必ず検知する
  • GIVINGを使うと元の値を変えずに計算できる
  • 仮想小数点の項目は計算後に桁位置が自動調整される
🔀 5. 条件分岐（IF・EVALUATE） ✅ ▼

条件分岐はIF文とEVALUATE文の2種類。複数条件には必ずEVALUATEを使いましょう。

📌 IF文の基本

*基本形
IF WS-SHIYO > 300
    MOVE "高圧" TO WS-KUBUN
END-IF

*ELSE付き
IF WS-RYOKIN >= 10000
    MOVE "Y" TO WS-KOUGAKU-FLG
ELSE
    MOVE "N" TO WS-KOUGAKU-FLG
END-IF

*ネスト（深くなりすぎに注意！）
IF WS-STATUS = "A"
    IF WS-SHIYO > 0
        PERFORM 正常処理
    ELSE
        PERFORM 使用量ゼロ処理
    END-IF
ELSE
    PERFORM エラー処理
END-IF

📌 条件の書き方

*比較演算子
= (EQUAL)  > (GREATER)  < (LESS)
>= (NOT LESS)  <= (NOT GREATER)  <> (NOT EQUAL)

*論理演算子（AND/OR/NOT）
IF WS-A > 0 AND WS-B > 0 ...
IF WS-A = 0 OR WS-B = 0 ...
IF NOT WS-EOF = "Y" ...

*88レベル条件名（可読性が高い！）
IF ファイル終了 ...       *WS-EOF-FLG = "Y"
IF 高額顧客 ...           *WS-RYOKIN >= 10000
IF NOT エラー ...

*数値範囲チェック
IF WS-SHIYO >= 0 AND WS-SHIYO <= 9999 ...

*文字チェック
IF WS-CODE IS NUMERIC ...     *全桁数字か
IF WS-NAME IS ALPHABETIC ...  *英字のみか

📌 EVALUATE文（多岐分岐）

*パターン1：変数で分岐
EVALUATE WS-KEIYAKU-KBN
    WHEN "A"    PERFORM 低圧処理
    WHEN "B"    PERFORM 中圧処理
    WHEN "C"    PERFORM 高圧処理
    WHEN SPACES PERFORM 未設定処理
    WHEN OTHER  PERFORM 異常処理
END-EVALUATE

*パターン2：EVALUATE TRUE（条件で分岐・実務で最頻出！）
EVALUATE TRUE
    WHEN WS-SHIYO = 0
        PERFORM ゼロ使用処理
    WHEN WS-SHIYO <= 120
        PERFORM 第一段階処理
    WHEN WS-SHIYO <= 300
        PERFORM 第二段階処理
    WHEN OTHER
        PERFORM 第三段階処理
END-EVALUATE

*パターン3：複数条件の組み合わせ
EVALUATE WS-TYPE ALSO WS-STATUS
    WHEN "A" ALSO "1"  PERFORM 処理AA
    WHEN "A" ALSO "2"  PERFORM 処理AB
    WHEN "B" ALSO ANY  PERFORM 処理B全般
    WHEN OTHER         PERFORM 異常処理
END-EVALUATE

📌 実務ポイント

  • IFのネストは3段階まで。それ以上はEVALUATEに切り替える
  • OTHERは必ず書く（想定外入力の救済）
  • 88レベルを活用すると条件文が自然言語に近くなる
  • END-IF・END-EVALUATEは必ず対応させる（スコープ終端）
🔄 6. 繰り返し処理（PERFORM） ✅ ▼

PERFORMはCOBOLの繰り返しの要。段落呼び出し・回数指定・条件ループ・VARYINGを使いこなしましょう。

📌 段落呼び出し

*単純呼び出し（サブルーチン的に使う）
PERFORM 初期処理
PERFORM 入力処理
PERFORM 集計処理
PERFORM 出力処理

*段落の定義（A領域に書く）
初期処理.
    MOVE ZEROS TO WS-COUNTER
    MOVE ZEROS TO WS-GOUKEI
    .  ← ピリオドで段落終了

入力処理.
    OPEN INPUT 顧客ファイル
    .

📌 回数指定PERFORM

*固定回数
PERFORM 10 TIMES
    ADD 1 TO WS-COUNTER
END-PERFORM

*変数回数
PERFORM WS-COUNT TIMES
    PERFORM 明細出力処理
END-PERFORM

📌 条件付きPERFORM（ループ）

*UNTIL（条件が真になるまで繰り返す）
PERFORM UNTIL WS-EOF = "Y"
    READ 顧客ファイル
        AT END MOVE "Y" TO WS-EOF
    END-READ
    IF NOT WS-EOF = "Y"
        PERFORM 明細処理
    END-IF
END-PERFORM

*WITH TEST BEFORE（デフォルト：先に条件チェック）
PERFORM WITH TEST BEFORE UNTIL WS-COUNT > 100
    ADD 1 TO WS-COUNT
END-PERFORM

*WITH TEST AFTER（後判定：最低1回は実行）
PERFORM WITH TEST AFTER UNTIL WS-INPUT = "Q"
    PERFORM 入力処理
END-PERFORM

📌 PERFORM VARYING（カウンタ付き）

*配列処理に必須！
PERFORM VARYING WS-IDX FROM 1 BY 1 UNTIL WS-IDX > 12
    ADD 月別集計(WS-IDX) TO WS-NENKAN-GOUKEI
END-PERFORM

*多重ループ（表データ処理）
PERFORM VARYING WS-I FROM 1 BY 1 UNTIL WS-I > 3
    PERFORM VARYING WS-J FROM 1 BY 1 UNTIL WS-J > 12
        ADD 集計表(WS-I, WS-J) TO WS-GOUKEI
    END-PERFORM
END-PERFORM

📌 実務ポイント

  • ファイルループはUNTIL + AT ENDパターンが定石
  • 段落は「1つの処理単位」に絞ること（長すぎると保守困難）
  • PERFORM THRU は使わない（段落の独立性が崩れる）
  • インラインPERFORMとアウトラインPERFORMを適切に使い分ける
📂 7. ファイル操作（順次・索引・直接） ✅ ▼

COBOLの本領はファイル処理。順次・索引（ISAM）・直接の3種類を理解しましょう。

📌 ファイル定義の流れ（3ステップ）

*①ENVIRONMENT DIVISION: 名前と物理ファイルを紐付け
FILE-CONTROL.
    SELECT 顧客マスタ
        ASSIGN TO "KOKYAKU.DAT"
        ORGANIZATION IS INDEXED          *索引ファイル
        ACCESS MODE IS DYNAMIC           *順次＋直接両方
        RECORD KEY IS KM-KOKYAKU-ID      *主キー
        ALTERNATE RECORD KEY IS KM-NAME  *副キー（重複可）
            WITH DUPLICATES.

*②DATA DIVISION: レコード構造を定義
FILE SECTION.
FD  顧客マスタ.
01  顧客マスタレコード.
    05 KM-KOKYAKU-ID  PIC X(8).
    05 KM-NAME        PIC X(20).
    05 KM-SHIYO       PIC 9(6).
    05 KM-RYOKIN      PIC 9(9).

*③WORKING-STORAGE: EOF管理フラグ
77 WS-EOF           PIC X(1) VALUE "N".
   88 ファイル終了   VALUE "Y".

📌 順次ファイルの定型パターン

*【読み込み】
OPEN INPUT 顧客マスタ
PERFORM UNTIL ファイル終了
    READ 顧客マスタ
        AT END     SET ファイル終了 TO TRUE
        NOT AT END PERFORM 顧客処理
    END-READ
END-PERFORM
CLOSE 顧客マスタ

*【書き出し】
OPEN OUTPUT 出力ファイル
PERFORM VARYING WS-IDX FROM 1 BY 1 UNTIL WS-IDX > WS-CNT
    MOVE 出力データ(WS-IDX) TO 出力レコード
    WRITE 出力レコード
END-PERFORM
CLOSE 出力ファイル

*【追記】
OPEN EXTEND 出力ファイル
WRITE 出力レコード
CLOSE 出力ファイル

📌 索引ファイル（ISAM）のCRUD

*【キー検索・READ】
MOVE "A0000001" TO KM-KOKYAKU-ID
READ 顧客マスタ
    KEY IS KM-KOKYAKU-ID
    INVALID KEY PERFORM キー未存在処理
    NOT INVALID KEY PERFORM 顧客表示処理
END-READ

*【新規追加・WRITE】
MOVE "A0000099" TO KM-KOKYAKU-ID
MOVE "新規顧客"  TO KM-NAME
WRITE 顧客マスタレコード
    INVALID KEY PERFORM キー重複エラー処理
END-WRITE

*【更新・REWRITE】
READ 顧客マスタ ...  *先にREADする
ADD 100 TO KM-SHIYO
REWRITE 顧客マスタレコード
    INVALID KEY PERFORM 更新エラー処理
END-REWRITE

*【削除・DELETE】
READ 顧客マスタ ...  *先にREADする
DELETE 顧客マスタ RECORD
    INVALID KEY PERFORM 削除エラー処理
END-DELETE

📌 ファイル状態（FILE STATUS）チェック

*ファイル状態を監視する（実務では必須！）
FILE-CONTROL.
    SELECT 顧客マスタ
        ASSIGN TO "KOKYAKU.DAT"
        FILE STATUS IS WS-FILE-STATUS.

WORKING-STORAGE SECTION.
01 WS-FILE-STATUS  PIC X(2).

*主なステータス値
*"00" : 正常
*"10" : ファイル終了（AT ENDと同じ）
*"22" : キー重複
*"23" : キー未存在
*"35" : ファイル未存在
*"91" : ファイルロックエラー

READ 顧客マスタ ...
IF WS-FILE-STATUS NOT = "00" AND NOT = "10"
    PERFORM ファイルエラー処理
END-IF
🗂️ 8. 表（テーブル）処理・OCCURS ✅ ▼

OCCURSで配列（テーブル）を定義します。月別集計・マスタテーブルのメモリ展開など実務で必須のスキルです。

📌 基本的なOCCURSの定義

*1次元テーブル（12ヶ月分の集計）
01 WS-TSUKI-GOUKEI-TBL.
   05 WS-TSUKI-GOUKEI OCCURS 12 TIMES
                      PIC 9(10).
*使い方
ADD WS-RYOKIN TO WS-TSUKI-GOUKEI(WS-TSUKI)
MOVE WS-TSUKI-GOUKEI(1) TO WS-DISP  *1月分

*複数項目のOCCURS
01 WS-MEISAI-TBL.
   05 WS-MEISAI OCCURS 100 TIMES.
      10 WS-MEISAI-ID    PIC X(8).
      10 WS-MEISAI-NM    PIC X(20).
      10 WS-MEISAI-AMT   PIC 9(9).

*テーブルへのデータセット
MOVE "A001"    TO WS-MEISAI-ID(1)
MOVE "電力基本料" TO WS-MEISAI-NM(1)
MOVE 1500      TO WS-MEISAI-AMT(1)

📌 OCCURS DEPENDING ON（可変長テーブル）

01 WS-DATA-CNT     PIC 9(3).
01 WS-DATA-TBL.
   05 WS-DATA OCCURS 1 TO 100 TIMES
              DEPENDING ON WS-DATA-CNT
              PIC 9(9).

*件数に応じてメモリを動的に使う

📌 INDEXED BY（インデックス付きテーブル）

01 WS-RYOKIN-TBL.
   05 WS-RYOKIN-ENTRY OCCURS 12 TIMES
                      INDEXED BY WS-IDX
                      PIC 9(10).

*SET命令でインデックス操作
SET WS-IDX TO 1
PERFORM UNTIL WS-IDX > 12
    ADD WS-RYOKIN-ENTRY(WS-IDX) TO WS-TOTAL
    SET WS-IDX UP BY 1
END-PERFORM

*SEARCH（線形検索）
SEARCH WS-RYOKIN-ENTRY
    AT END PERFORM 未検出処理
    WHEN WS-RYOKIN-ENTRY(WS-IDX) > 100000
        PERFORM 高額処理
END-SEARCH

*SEARCH ALL（二分探索・ASCENDING KEY指定が必要）
SEARCH ALL WS-RYOKIN-ENTRY
    AT END PERFORM 未検出処理
    WHEN WS-RYOKIN-ENTRY(WS-IDX) = WS-TARGET
        PERFORM 検出処理
END-SEARCH

📌 実務ポイント

  • テーブルの添字は1始まり（0ではない！）
  • 添字が上限を超えるとメモリ破壊になるので範囲チェック必須
  • 大量データはファイルから読んでテーブルに展開する「テーブルロード」パターンが多い
  • 2次元テーブルはOCCURS OCCURSでネスト定義できる
📝 9. 文字列処理 ✅ ▼

帳票・メッセージ生成に必須の文字列操作。STRING・UNSTRING・INSPECT・REFERENCE MODIFICATIONを覚えましょう。

📌 STRING（文字列結合）

*複数項目を1つに結合
01 WS-MSG   PIC X(50).
01 WS-NAME  PIC X(10) VALUE "田中一郎".
01 WS-MONTH PIC X(4)  VALUE "10月".

STRING WS-NAME DELIMITED SIZE
       "様 " DELIMITED SIZE
       WS-MONTH DELIMITED SIZE
       "分の料金通知です"  DELIMITED SIZE
       INTO WS-MSG
END-STRING
*→ WS-MSG = "田中一郎様 10月分の料金通知です"

*文字数カウント付き
STRING ... INTO WS-MSG WITH POINTER WS-PTR
*→ WS-PTRに結合後の次の位置が入る

📌 UNSTRING（文字列分割）

*区切り文字で分割
01 WS-INPUT PIC X(30) VALUE "A001,田中,100".
01 WS-F1    PIC X(5).
01 WS-F2    PIC X(10).
01 WS-F3    PIC X(10).

UNSTRING WS-INPUT DELIMITED BY ","
    INTO WS-F1 WS-F2 WS-F3
END-UNSTRING
*→ WS-F1="A001" WS-F2="田中" WS-F3="100"

*カウント取得
UNSTRING WS-INPUT DELIMITED BY ","
    INTO WS-F1 COUNT IN WS-F1-LEN
         WS-F2 COUNT IN WS-F2-LEN
         WS-F3 COUNT IN WS-F3-LEN
    TALLYING IN WS-FIELD-CNT
END-UNSTRING

📌 INSPECT（文字検査・置換）

*文字のカウント
INSPECT WS-DATA
    TALLYING WS-CNT FOR ALL "A"      *"A"の個数
INSPECT WS-DATA
    TALLYING WS-CNT FOR LEADING " "  *先頭スペース数

*文字の置換
INSPECT WS-DATA
    REPLACING ALL " " BY "0"         *スペースを0に置換
INSPECT WS-DATA
    REPLACING LEADING "0" BY " "     *先頭ゼロをスペースに

*カウントと置換を同時に
INSPECT WS-DATA
    TALLYING WS-CNT FOR ALL "/"
    REPLACING ALL "/" BY "-"

📌 REFERENCE MODIFICATION（部分参照）

*文字列の一部を切り出す  書式: 変数名(開始位置:長さ)
01 WS-DATE  PIC X(8) VALUE "20241015".

*年・月・日を切り出す
MOVE WS-DATE(1:4) TO WS-YEAR    *"2024"
MOVE WS-DATE(5:2) TO WS-MONTH   *"10"
MOVE WS-DATE(7:2) TO WS-DAY     *"15"

*動的な切り出し
MOVE WS-INPUT(WS-START:WS-LEN) TO WS-WORK

*実務例：伝票番号から年度を取得
01 WS-DENPYO-NO  PIC X(12) VALUE "2024-001-001".
MOVE WS-DENPYO-NO(1:4) TO WS-NENDO  *"2024"
🔧 10. サブプログラムとCALL文 ✅ ▼

大規模システムでは処理をサブプログラムに分割します。CALL文でプログラム間の連携を実装します。

📌 CALL文の基本

*呼び出し側（主プログラム）
WORKING-STORAGE SECTION.
01 WS-PARAM.
   05 WS-SHIYO      PIC 9(6).
   05 WS-RYOKIN     PIC 9(9).
   05 WS-RETURN-CD  PIC 9(2).

PROCEDURE DIVISION.
    MOVE 250 TO WS-SHIYO
    CALL "RYOKIN01" USING WS-PARAM
    IF WS-RETURN-CD NOT = 0
        PERFORM エラー処理
    END-IF

*静的CALL（コンパイル時に結合）
CALL "RYOKIN01" USING WS-PARAM

*動的CALL（実行時に結合）
MOVE "RYOKIN01" TO WS-PROG-NAME
CALL WS-PROG-NAME USING WS-PARAM

📌 サブプログラム側

*呼び出される側（サブプログラム）
IDENTIFICATION DIVISION.
PROGRAM-ID. RYOKIN01.

DATA DIVISION.
LINKAGE SECTION.                  *受け取るパラメータはここに定義
01 LS-PARAM.
   05 LS-SHIYO      PIC 9(6).
   05 LS-RYOKIN     PIC 9(9).
   05 LS-RETURN-CD  PIC 9(2).

PROCEDURE DIVISION USING LS-PARAM.
    MOVE 0 TO LS-RETURN-CD

    EVALUATE TRUE
        WHEN LS-SHIYO <= 120
            COMPUTE LS-RYOKIN ROUNDED = LS-SHIYO * 17.78
        WHEN LS-SHIYO <= 300
            COMPUTE LS-RYOKIN ROUNDED
                = 120 * 17.78 + (LS-SHIYO - 120) * 22.86
        WHEN OTHER
            COMPUTE LS-RYOKIN ROUNDED
                = 120 * 17.78 + 180 * 22.86
                + (LS-SHIYO - 300) * 25.72
        END-EVALUATE

    GOBACK.  *← CALLで呼ばれた場合はSTOP RUNではなくGOBACK

📌 BY REFERENCE / BY CONTENT

*BY REFERENCE（デフォルト）：アドレスを渡す → 呼び出し側の値も変わる
CALL "PROG" USING BY REFERENCE WS-DATA

*BY CONTENT：値のコピーを渡す → 呼び出し側は変わらない（安全）
CALL "PROG" USING BY CONTENT WS-DATA

*混在も可能
CALL "RYOKIN01" USING BY REFERENCE WS-IN-DATA
                      BY CONTENT   WS-CONST-DATA

📌 実務ポイント

  • 業務ロジック（料金計算など）は必ずサブプログラムに切り出す
  • LINKAGE SECTIONとUSING句の項目順・型を呼び出し側と必ず合わせる
  • 戻りコード（RETURN-CODE）で正常/エラーを呼び出し側に伝える
  • GOBACK はプログラムを終了せず呼び出し元に戻る（STOP RUNとの違い）
🧪 11. テスト設計の基礎と技法 ✅ ▼

単体テストを効率よく・漏れなく設計するための技法を学びます。

📌 単体テストとは

  • プログラムの最小単位（段落・モジュール）が仕様通り動くかを確認
  • バグを早期に発見 → 後工程の手戻りコストを最小化
  • COBOLでは「段落」単位がテストの基本単位

📌 同値分割法

【仕様】契約種別を判定する
 使用量  0〜120kWh  → "低圧A"
 使用量 121〜300kWh → "低圧B"
 使用量 301kWh〜   → "高圧"
 使用量 ＜0        → エラー

【同値クラス】
 有効: ①0〜120　②121〜300　③301〜
 無効: ④負値

【テストケース（各クラスから1つ）】
TC-01: 60kWh  → "低圧A"  ①の代表値
TC-02: 200kWh → "低圧B"  ②の代表値
TC-03: 500kWh → "高圧"   ③の代表値
TC-04: -1kWh  → エラー   ④の代表値

📌 境界値分析

【境界値は同値クラスの端の値±1】
TC-05:   0kWh → "低圧A"  ①の下限
TC-06: 120kWh → "低圧A"  ①の上限 ← バグが出やすい！
TC-07: 121kWh → "低圧B"  ②の下限 ← バグが出やすい！
TC-08: 300kWh → "低圧B"  ②の上限 ← バグが出やすい！
TC-09: 301kWh → "高圧"   ③の下限 ← バグが出やすい！

【なぜ境界値にバグが多い？】
IF WS-SHIYO >= 121 ...  ← > と >= を間違えやすい
IF WS-SHIYO < 300 ...   ← <= と < を間違えやすい

📌 デシジョンテーブル（複数条件の組み合わせ）

【仕様】割引判定
 ・高齢者（65歳以上）→ 10%割引
 ・低所得世帯       → 5%割引
 ・両方該当         → 15%割引
 ・どちらでもない   → 割引なし

【デシジョンテーブル】
条件              | R1 | R2 | R3 | R4
高齢者か          |  Y |  Y |  N |  N
低所得世帯か      |  Y |  N |  Y |  N
─────────────────+────+────+────+────
アクション        |    |    |    |
15%割引適用       |  ✓ |    |    |
10%割引適用       |    |  ✓ |    |
5%割引適用        |    |    |  ✓ |
割引なし          |    |    |    |  ✓

【テストケース: 4パターン全て必要】
TC-10: 高齢者+低所得 → 15%割引
TC-11: 高齢者のみ   → 10%割引
TC-12: 低所得のみ   → 5%割引
TC-13: どちらでもない → 割引なし

📌 ホワイトボックステスト

【C0カバレッジ（命令網羅）】
→ 全命令を最低1回実行

【C1カバレッジ（分岐網羅）】
→ 全分岐（TRUEとFALSEの両方）を実行
　IFが3つあれば最低2^3=8通り必要

【実務では通常C1を目標とする】

例：
IF WS-A > 0         ← TRUEとFALSEの両方
    IF WS-B > 0     ← TRUEとFALSEの両方
        PERFORM 処理1
    ELSE
        PERFORM 処理2
    END-IF
ELSE
    PERFORM 処理3
END-IF
→ テストケース: 4パターン必要

【1】桁あふれチェック

  → ファイル0件（空ファイル）
  → ファイル1件のみ
  → ファイル最大件数
  → ファイル未存在（FILE STATUS = "35"）

  → 送り側の方が桁数が大きい場合に上位桁が消えるか
  → 仮想小数点のずれ

【4】ループ処理
  → 0回ループ（条件が最初から偽）
  → 1回のみ実行
  → 最大件数での実行

【5】日付・期間処理
  → 月末日（28/29/30/31日）
  → 年をまたぐ期間
  → うるう年（2月29日）

📌 テスト結果の記録と報告

  • NGの場合は実際の出力値を必ず記録する
  • バグ票には再現手順・期待値・実際値を明確に記載
  • 修正後は回帰テスト（関連テストの再実行）を行う

🔨 3. コンパイル・ビルドの操作 ✅ ▼

ビルドエラーの読み方と対処法を身につけると、エラー解消の時間が劇的に短くなります。

📌 ビルド手順

【方法1: 自動ビルド（推奨）】
プロジェクト → 自動的にビルド にチェック
→ ファイル保存時に自動コンパイル
→ エラーがあればすぐに赤マーカーで表示

【方法2: 手動ビルド】
Ctrl + B または プロジェクト → プロジェクトのビルド
→ コンソールビューにビルドログが出力される

【方法3: クリーンビルド（エラーが解消しないとき）】
プロジェクト → クリーン
→ 全オブジェクトを削除して最初からビルド
→ 「コンパイル済みなのに古いエラーが消えない」場合に有効

📌 エラーメッセージの読み方

【典型的なエラー例】
PROG001.cob(45): JMP0001E-S  ...
       ↑             ↑        ↑
    ファイル名      行番号   エラーコード

【重要度の見方】
-E : エラー（ビルド失敗・必ず修正）
-W : 警告（ビルドは通るが要確認）
-I : 情報（参考情報）

【よくあるエラーコードと対処】
JMP0001E : 未定義の変数名 → DATA DIVISIONの定義を確認
JMP0023E : PICTURE句の不正 → PIC指定子を確認
JMP0150E : 対応するEND-IFなし → IF文のネストを確認
JMP0200E : COPY文のファイルが見つからない → COPYLIBパス設定を確認
JMP0300W : データの切り捨て発生 → 受け取り側の桁数を確認

📌 COPYライブラリのパス設定

プロジェクト右クリック → プロパティ
→ NetCOBOLビルド設定 → コンパイルオプション
→ COPY文のライブラリパスを追加
→ 例: /home/user/copylib;/usr/local/cobol/copylib

* COPYファイルが見つからないエラーは9割これで解決

📌 デバッグ実行の開始手順

①【デバッグ対象のビルド設定確認】

【使い分け】

🗄️ 1. SQLとデータベースの基礎 ✅ ▼

SQLはStructured Query Languageの略。リレーショナルDBのデータを操作するための言語です。

📌 RDBの基本概念

【テーブル（表）】
データを行（レコード）と列（カラム）で管理する

例：顧客テーブル（CUSTOMER）
┌──────────┬──────────┬──────────┬──────────┐
│ CUST_ID  │ CUST_NAME│ AREA_CD  │ CONTRACT │
├──────────┼──────────┼──────────┼──────────┤
│ C0000001 │ 田中一郎  │ 01       │ A        │
│ C0000002 │ 鈴木花子  │ 02       │ B        │
│ C0000003 │ 佐藤次郎  │ 01       │ A        │
└──────────┴──────────┴──────────┴──────────┘

主キー（Primary Key）: CUST_ID → 重複不可・NULL不可
外部キー（Foreign Key）: 他テーブルの主キーを参照

📌 SQLの種類

【DML: データ操作言語】
SELECT  → データを検索・取得
INSERT  → データを追加
UPDATE  → データを更新
DELETE  → データを削除

【DDL: データ定義言語】
CREATE TABLE  → テーブル作成
ALTER TABLE   → テーブル変更
DROP TABLE    → テーブル削除

【DCL: データ制御言語】
COMMIT    → 変更を確定（永続化）
ROLLBACK  → 変更を取り消し
GRANT     → 権限付与

📌 COBOLとSQLの連携（EXEC SQL）

*COBOLプログラム内でSQLを使う場合
EXEC SQL
    SELECT CUST_NAME
    INTO   :WS-CUST-NAME
    FROM   CUSTOMER
    WHERE  CUST_ID = :WS-CUST-ID
END-EXEC

*:（コロン）付きはCOBOL変数（ホスト変数）
*SQLCODE で実行結果を確認
IF SQLCODE = 0
    DISPLAY "取得成功: " WS-CUST-NAME
ELSE IF SQLCODE = 100
    DISPLAY "データなし"
ELSE
    PERFORM SQLエラー処理
END-IF

📌 主なSQLCODE値

  0  : 正常終了
100  : データなし（NOT FOUND）
-803 : 主キー重複（INSERT失敗）
-805 : パッケージ未存在
-911 : デッドロック / タイムアウト
-922 : 権限なし
🔍 2. SELECT基本（検索の基礎） ✅ ▼

SELECTはSQLの核心。どの列を・どのテーブルから・どの条件で取得するかを覚えましょう。

📌 SELECT基本構文

-- 全列取得（本番では非推奨）
SELECT * FROM CUSTOMER;

-- 列を指定して取得
SELECT CUST_ID, CUST_NAME, CONTRACT
FROM   CUSTOMER;

-- 列に別名（エイリアス）をつける
SELECT CUST_ID    AS 顧客ID,
       CUST_NAME  AS 顧客名,
       CONTRACT   AS 契約種別
FROM   CUSTOMER;

-- 重複を除外
SELECT DISTINCT AREA_CD FROM CUSTOMER;

-- 計算列
SELECT CUST_ID,
       SHIYO_KWH,
       SHIYO_KWH * 17.78 AS 概算料金
FROM   USAGE_DATA;

📌 WHERE句（条件指定）

-- 等値比較
SELECT * FROM CUSTOMER WHERE CONTRACT = 'A';

-- 範囲指定
SELECT * FROM USAGE_DATA WHERE SHIYO_KWH >= 300;
SELECT * FROM USAGE_DATA
WHERE  SHIYO_KWH BETWEEN 121 AND 300;

-- リスト一致
SELECT * FROM CUSTOMER
WHERE  AREA_CD IN ('01', '02', '03');

-- パターンマッチ（LIKE）
SELECT * FROM CUSTOMER WHERE CUST_NAME LIKE '田中%';
-- % : 任意の文字列  _ : 任意の1文字

-- NULL判定（= NULLはNG！）
SELECT * FROM CUSTOMER WHERE TEL IS NULL;
SELECT * FROM CUSTOMER WHERE TEL IS NOT NULL;

-- 複合条件
SELECT * FROM CUSTOMER
WHERE  CONTRACT = 'A'
AND    AREA_CD = '01';

SELECT * FROM CUSTOMER
WHERE  CONTRACT = 'A'
OR     CONTRACT = 'B';

📌 ORDER BY（並び替え）

-- 昇順（デフォルト）
SELECT * FROM CUSTOMER ORDER BY CUST_NAME ASC;

-- 降順
SELECT * FROM USAGE_DATA ORDER BY SHIYO_KWH DESC;

-- 複数キー
SELECT * FROM USAGE_DATA
ORDER BY AREA_CD ASC, SHIYO_KWH DESC;

📌 実務ポイント

  • SELECT * は本番NGが多い（列追加時に意図しない動作になる）
  • NULLとの比較は必ずIS NULL/IS NOT NULLを使う
  • 文字列はシングルクォート''で囲む（ダブルクォートは列名）
📊 3. 集計関数とGROUP BY ✅ ▼

集計関数でデータを集約し、GROUP BYでグループ別の集計ができます。HAVINGとの組み合わせも重要。

📌 集計関数

-- COUNT: 件数
SELECT COUNT(*)       FROM CUSTOMER;          -- 全行数
SELECT COUNT(TEL)     FROM CUSTOMER;          -- NULLを除いた件数
SELECT COUNT(DISTINCT AREA_CD) FROM CUSTOMER; -- ユニーク件数

-- SUM / AVG / MAX / MIN
SELECT SUM(SHIYO_KWH)  AS 合計使用量,
       AVG(SHIYO_KWH)  AS 平均使用量,
       MAX(SHIYO_KWH)  AS 最大使用量,
       MIN(SHIYO_KWH)  AS 最小使用量
FROM   USAGE_DATA
WHERE  YEAR_MONTH = '202410';

📌 GROUP BY（グループ集計）

-- エリア別の顧客数と平均使用量
SELECT AREA_CD,
       COUNT(*)            AS 顧客数,
       AVG(SHIYO_KWH)      AS 平均使用量,
       SUM(SHIYO_KWH)      AS 合計使用量
FROM   USAGE_DATA
WHERE  YEAR_MONTH = '202410'
GROUP BY AREA_CD
ORDER BY AREA_CD;

-- 複数列でGROUP BY
SELECT AREA_CD, CONTRACT,
       COUNT(*) AS 件数
FROM   CUSTOMER
GROUP BY AREA_CD, CONTRACT;

📌 HAVING（グループへの条件）

-- WHEREとHAVINGの違い
-- WHERE  : 集計前のデータに条件
-- HAVING : 集計後のグループに条件

-- 平均使用量が200kWh以上のエリアのみ
SELECT AREA_CD,
       AVG(SHIYO_KWH) AS 平均使用量
FROM   USAGE_DATA
WHERE  YEAR_MONTH = '202410'    -- ①まず月でフィルタ
GROUP BY AREA_CD
HAVING AVG(SHIYO_KWH) >= 200    -- ②集計後にフィルタ
ORDER BY 平均使用量 DESC;

📌 SELECT文の実行順序（重要！）

①FROM     テーブルを取得
②WHERE    行をフィルタ
③GROUP BY グループ化
④HAVING   グループをフィルタ
⑤SELECT   列を選択・計算
⑥ORDER BY 並び替え
⑦LIMIT    件数制限

※ この順序を理解するとエラーの原因が分かる！
🔗 4. テーブル結合（JOIN） ✅ ▼

JOINは複数テーブルを結合して一度に取得する強力な機能。INNER/LEFT/RIGHTの違いを確実に理解しましょう。

📌 INNER JOIN（内部結合・最頻出）

-- 両テーブルに一致するデータのみ取得
-- 顧客テーブルと使用量テーブルを結合
SELECT C.CUST_ID,
       C.CUST_NAME,
       U.YEAR_MONTH,
       U.SHIYO_KWH
FROM   CUSTOMER    C          -- エイリアスCをつける
INNER JOIN USAGE_DATA U        -- エイリアスU
ON     C.CUST_ID = U.CUST_ID
WHERE  U.YEAR_MONTH = '202410'
ORDER BY C.CUST_ID;

-- 結果: 使用量データが存在する顧客だけ表示

📌 LEFT JOIN（左外部結合）

-- 左テーブル（FROM側）は全件取得
-- 右テーブルに一致なければNULL
SELECT C.CUST_ID,
       C.CUST_NAME,
       U.SHIYO_KWH       -- 使用量なし → NULL
FROM   CUSTOMER    C
LEFT JOIN USAGE_DATA U
ON     C.CUST_ID = U.CUST_ID
AND    U.YEAR_MONTH = '202410';

-- 結果: 全顧客を取得（使用量がない顧客はNULL）

-- 未使用顧客だけ抽出（左結合 + IS NULL）
SELECT C.CUST_ID, C.CUST_NAME
FROM   CUSTOMER    C
LEFT JOIN USAGE_DATA U
ON     C.CUST_ID = U.CUST_ID
AND    U.YEAR_MONTH = '202410'
WHERE  U.CUST_ID IS NULL;   -- 右側がNULL = 未使用

📌 複数テーブルの結合

-- 顧客 × 使用量 × エリアマスタ
SELECT C.CUST_NAME,
       A.AREA_NAME,
       U.SHIYO_KWH,
       U.RYOKIN
FROM   CUSTOMER    C
INNER JOIN AREA_MST    A ON C.AREA_CD  = A.AREA_CD
INNER JOIN USAGE_DATA  U ON C.CUST_ID  = U.CUST_ID
WHERE  U.YEAR_MONTH = '202410';

📌 JOIN選択の判断基準

INNER JOIN → 両方に存在するデータだけ必要
LEFT  JOIN → 左テーブル全件 + 右は任意
RIGHT JOIN → 右テーブル全件 + 左は任意（LEFT JOINで書き換え可）
FULL  JOIN → 両テーブル全件（一方だけのデータもNULLで表示）

【実務では INNER JOIN と LEFT JOIN の2択がほとんど】
✏️ 5. データ更新（INSERT・UPDATE・DELETE） ✅ ▼

データの追加・更新・削除。WHERE句の指定漏れは大事故になるため、必ず確認してから実行します。

🔄 6. サブクエリ ✅ ▼

クエリの中にクエリを書く「サブクエリ」。WHERE・FROM・SELECTの中で使えます。

📌 WHERE句のサブクエリ

-- 平均以上の使用量を持つ顧客
SELECT CUST_ID, SHIYO_KWH
FROM   USAGE_DATA
WHERE  YEAR_MONTH = '202410'
AND    SHIYO_KWH > (
    SELECT AVG(SHIYO_KWH)
    FROM   USAGE_DATA
    WHERE  YEAR_MONTH = '202410'
);

-- IN句でのサブクエリ
SELECT CUST_NAME FROM CUSTOMER
WHERE  CUST_ID IN (
    SELECT CUST_ID FROM USAGE_DATA
    WHERE  SHIYO_KWH >= 300
);

-- EXISTS（存在確認・パフォーマンスが良い）
SELECT C.CUST_NAME
FROM   CUSTOMER C
WHERE  EXISTS (
    SELECT 1 FROM USAGE_DATA U
    WHERE  U.CUST_ID    = C.CUST_ID
    AND    U.SHIYO_KWH  >= 300
);

📌 FROM句のサブクエリ（インラインビュー）

-- 月別集計をさらに絞り込む
SELECT *
FROM (
    SELECT AREA_CD,
           COUNT(*)         AS 件数,
           AVG(SHIYO_KWH)   AS 平均使用量
    FROM   USAGE_DATA
    WHERE  YEAR_MONTH = '202410'
    GROUP BY AREA_CD
) AS 月別集計
WHERE 平均使用量 >= 200;

📌 WITH句（共通テーブル式・CTE）

-- 可読性が高く実務で多用される！
WITH 月別集計 AS (
    SELECT AREA_CD,
           AVG(SHIYO_KWH) AS 平均使用量
    FROM   USAGE_DATA
    WHERE  YEAR_MONTH = '202410'
    GROUP BY AREA_CD
),
高使用エリア AS (
    SELECT AREA_CD
    FROM   月別集計
    WHERE  平均使用量 >= 200
)
SELECT C.CUST_ID, C.CUST_NAME
FROM   CUSTOMER C
WHERE  C.AREA_CD IN (SELECT AREA_CD FROM 高使用エリア);

⚡ 8. ウィンドウ関数（分析関数） ✅ ▼

ウィンドウ関数はGROUP BYを使わずに集計しながら元の行を保持できる強力な機能です。

📌 ROW_NUMBER・RANK・DENSE_RANK

-- エリア内での使用量ランキング
SELECT CUST_ID,
       AREA_CD,
       SHIYO_KWH,
       ROW_NUMBER() OVER (
           PARTITION BY AREA_CD      -- エリア別に区切る
           ORDER BY SHIYO_KWH DESC   -- 使用量降順
       ) AS 行番号,
       RANK() OVER (
           PARTITION BY AREA_CD
           ORDER BY SHIYO_KWH DESC
       ) AS 順位           -- 同率は同じ順位、次をスキップ
FROM   USAGE_DATA
WHERE  YEAR_MONTH = '202410';

--【ROW_NUMBERとRANKの違い】
-- 使用量: 300, 300, 200 の場合
-- ROW_NUMBER: 1, 2, 3（重複でも連番）
-- RANK:       1, 1, 3（同率は同じ順位、2はスキップ）
-- DENSE_RANK: 1, 1, 2（同率は同じ順位、スキップなし）

📌 集計ウィンドウ関数

-- 累計・移動平均・前月比
SELECT YEAR_MONTH,
       SHIYO_KWH,
       -- 累計使用量
       SUM(SHIYO_KWH) OVER (
           ORDER BY YEAR_MONTH
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS 累計使用量,
       -- 直近3ヶ月移動平均
       AVG(SHIYO_KWH) OVER (
           ORDER BY YEAR_MONTH
           ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
       ) AS 3ヶ月移動平均
FROM   USAGE_DATA
WHERE  CUST_ID = 'C0000001'
ORDER BY YEAR_MONTH;

📌 LAG・LEAD（前後の行を参照）

-- 前月比較
SELECT YEAR_MONTH,
       SHIYO_KWH,
       LAG(SHIYO_KWH, 1) OVER (
           PARTITION BY CUST_ID
           ORDER BY YEAR_MONTH
       ) AS 前月使用量,
       SHIYO_KWH - LAG(SHIYO_KWH, 1) OVER (
           PARTITION BY CUST_ID
           ORDER BY YEAR_MONTH
       ) AS 前月比増減
FROM   USAGE_DATA
WHERE  CUST_ID = 'C0000001'
ORDER BY YEAR_MONTH;
🚀 9. パフォーマンスとインデックス ✅ ▼

大量データを扱う業務システムではSQL性能が重要課題です。インデックスとチューニングの基礎を学びます。

📌 インデックスとは

-- インデックスなし: 全行スキャン（フルスキャン）
-- インデックスあり: B木構造で高速検索

-- 単一インデックス
CREATE INDEX IDX_USAGE_CUST
    ON USAGE_DATA (CUST_ID);

-- 複合インデックス（列の順番が重要！）
CREATE INDEX IDX_USAGE_AREA_MONTH
    ON USAGE_DATA (AREA_CD, YEAR_MONTH);
-- ↑ AREA_CDだけでもOK、YEAR_MONTHだけではNG

-- ユニークインデックス
CREATE UNIQUE INDEX IDX_CUST_PK
    ON CUSTOMER (CUST_ID);

📌 インデックスが効かないパターン（要注意！）

-- ❌ 列に関数を使う
WHERE SUBSTRING(CUST_ID, 1, 3) = 'C00'
→ ✅ WHERE CUST_ID LIKE 'C00%'

-- ❌ 否定条件（NOT IN, !=, <>）
WHERE CONTRACT != 'A'
→ ✅ WHERE CONTRACT IN ('B', 'C') に書き換え

-- ❌ OR条件（インデックスが使われにくい）
WHERE CUST_ID = 'X' OR AREA_CD = '01'
→ ✅ UNION ALL に分割

-- ❌ 暗黙の型変換
WHERE CUST_ID = 1234    -- CUST_IDがCHAR型なのに数値
→ ✅ WHERE CUST_ID = 'C0001234'

-- ❌ LIKE前方一致以外
WHERE CUST_NAME LIKE '%田中'   -- 後方一致はNG
WHERE CUST_NAME LIKE '%田中%'  -- 中間一致もNG

📌 実行計画の確認

-- EXPLAIN で実行計画を確認（DB2）
EXPLAIN ALL FOR
SELECT * FROM USAGE_DATA
WHERE  CUST_ID = 'C0000001';

-- 確認ポイント
-- TBSCAN（テーブルスキャン）→ 遅い → インデックス追加検討
-- IXSCAN（インデックススキャン）→ 速い
-- FETCH → インデックスでキー取得 → テーブルアクセス

📌 パフォーマンスチューニングの基本方針

  • WHERE条件の列に適切なインデックスを設定する
  • SELECT * を避け、必要な列だけ取得する
  • 結合条件のキー列に必ずインデックスを設定する
  • 大量データの処理は一括処理よりも分割コミットを検討
  • 統計情報を定期的に更新してオプティマイザが正しく動くようにする
🔒 10. COBOL組み込みSQL（実践） ✅ ▼

実務でのCOBOL×DB2組み込みSQLの定型パターンを完全マスターします。

📌 ホスト変数の宣言

WORKING-STORAGE SECTION.

*SQLCAの取り込み（必須）
EXEC SQL INCLUDE SQLCA END-EXEC.

*ホスト変数（:をつけてSQLで使う）
01 WS-CUST-ID      PIC X(8).
01 WS-CUST-NAME    PIC X(20).
01 WS-SHIYO        PIC 9(8)  COMP-3.
01 WS-RYOKIN       PIC 9(10) COMP-3.
01 WS-YEAR-MONTH   PIC X(6).
*NULL標識変数（NULLを受け取る場合に必要）
01 WS-SHIYO-IND    PIC S9(4) COMP.

📌 1行SELECT（単一行取得）

EXEC SQL
    SELECT CUST_NAME,
           SHIYO_KWH
    INTO   :WS-CUST-NAME,
           :WS-SHIYO :WS-SHIYO-IND   -- NULL標識
    FROM   USAGE_DATA
    WHERE  CUST_ID    = :WS-CUST-ID
    AND    YEAR_MONTH = :WS-YEAR-MONTH
END-EXEC

EVALUATE SQLCODE
    WHEN  0    PERFORM 正常処理
    WHEN  100  PERFORM データなし処理
    WHEN  OTHER PERFORM SQLエラー処理
END-EVALUATE

*NULL確認
IF WS-SHIYO-IND < 0
    MOVE 0 TO WS-SHIYO     *NULLなら0セット
END-IF

📌 カーソル処理（複数行取得）

*複数行取得はカーソルを使う（実務で最頻出！）

*①カーソル宣言（DATA DIVISION）
EXEC SQL
    DECLARE USAGE_CUR CURSOR FOR
        SELECT CUST_ID, SHIYO_KWH, RYOKIN
        FROM   USAGE_DATA
        WHERE  YEAR_MONTH = :WS-YEAR-MONTH
        ORDER BY CUST_ID
END-EXEC

*②PROCEDURE DIVISION での処理
    EXEC SQL OPEN USAGE_CUR END-EXEC
    IF SQLCODE NOT = 0
        PERFORM カーソルオープンエラー処理
        STOP RUN
    END-IF

    PERFORM UNTIL SQLCODE = 100
        EXEC SQL
            FETCH USAGE_CUR
            INTO :WS-CUST-ID,
                 :WS-SHIYO,
                 :WS-RYOKIN
        END-EXEC

        IF SQLCODE = 0
            PERFORM 明細処理
        END-IF
    END-PERFORM

    EXEC SQL CLOSE USAGE_CUR END-EXEC

📌 UPDATE・INSERT・DELETEの実装

*UPDATE
EXEC SQL
    UPDATE USAGE_DATA
    SET    RYOKIN      = :WS-RYOKIN,
           UPDATED_AT  = CURRENT TIMESTAMP
    WHERE  CUST_ID    = :WS-CUST-ID
    AND    YEAR_MONTH = :WS-YEAR-MONTH
END-EXEC
IF SQLCODE NOT = 0
    PERFORM SQL更新エラー処理
END-IF

*INSERT
EXEC SQL
    INSERT INTO USAGE_DATA
        (CUST_ID, YEAR_MONTH, SHIYO_KWH, RYOKIN)
    VALUES
        (:WS-CUST-ID, :WS-YEAR-MONTH,
         :WS-SHIYO,   :WS-RYOKIN)
END-EXEC

*DELETE
EXEC SQL
    DELETE FROM USAGE_DATA
    WHERE  CUST_ID    = :WS-CUST-ID
    AND    YEAR_MONTH = :WS-YEAR-MONTH
END-EXEC

📌 実務の鉄則

  • 全EXEC SQLの後は必ずSQLCODE確認する
  • カーソルはOPEN → FETCH → CLOSEの順を守る
  • COMMIT/ROLLBACKは必ず対になるよう設計する
  • SQLCODE=-911（デッドロック）はリトライ処理を実装する
  • 本番のUPDATE/DELETEは事前にSELECTで件数確認する
